# SDwC

Survey-driven project documentation generator optimized for AI-assisted development with Claude Code.

## What is SDwC?

SDwC automates the creation of `CLAUDE.md`, project docs, and coding skills from a single YAML survey. Developers fill in a structured template (`intake_template.yaml`), and SDwC generates a complete documentation package that enables Claude Code to understand project context from day one.

## Problem

Developers using Claude Code lack structured project documentation, leading to wasted time and ineffective AI collaboration. SDwC solves this: **fill in a survey once, get a complete AI-ready project documentation package instantly.**

## Architecture

| Service | Type | Stack | Responsibility |
|---------|------|-------|----------------|
| sdwc-api | Backend API | Python, FastAPI, Jinja2 | Validate intake YAML, render templates, deliver generated ZIP |
| sdwc-web | Web UI | TypeScript, React, Vite | Single-page interface for uploading YAML, previewing output, downloading ZIP |

**Pattern**: Monolith (two independent services) with layered internals.

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/v1/template` | Download blank intake_template.yaml |
| POST | `/api/v1/validate` | Validate intake_data.yaml |
| POST | `/api/v1/preview` | Preview generated file structure |
| POST | `/api/v1/generate` | Generate documentation ZIP |

## Getting Started

### Prerequisites

- Python 3.12+ with Poetry
- Node.js 20+ with npm

### sdwc-api (Backend)

```bash
cd sdwc-api
poetry install
poetry run uvicorn sdwc_api.main:app --reload
```

### sdwc-web (Frontend)

```bash
cd sdwc-web
npm install
npm run dev
```

## Key Features

- **34 framework support** via skill-templates
- **Jinja2 Template Engine** for YAML-to-docs conversion
- **RFC 7807** error format
- **YAML bomb prevention** (1MB limit, 5s timeout, safe_load)
- **Swagger documentation** auto-generated by FastAPI

## CI/CD

### CI - GitHub Actions

Each service has its own workflow, triggered by path-filtered PRs and pushes to `main`.

| Workflow | Trigger paths | Check steps | Docker push |
|----------|--------------|-------------|-------------|
| `ci-sdwc-api.yml` | `sdwc-api/**`, `.sdwc/**` | ruff, mypy, pytest | On push to main |
| `ci-sdwc-web.yml` | `sdwc-web/**` | eslint, tsc, prettier, vitest, build | On push to main |

- **PR to main**: runs `check` job (lint, typecheck, test)
- **Push to main**: runs `check` then `docker` job (build + push to GHCR)
- **Image tags**: `sha-<7char>` + `latest`
- **Registry**: `ghcr.io/seongmin15/sdwc/sdwc-api`, `ghcr.io/seongmin15/sdwc/sdwc-web`

### CD - ArgoCD (GitOps)

Deployment to k3s via ArgoCD watching the `infra/` manifests.

## Deployment

Both services deploy to k3s (Kubernetes) on WSL2, with images stored in GitHub Container Registry (GHCR).

### Local Deployment (k3d)

Run the full stack locally using [k3d](https://k3d.io) (k3s-in-Docker).

**Prerequisites**: Docker Desktop running, [k3d](https://k3d.io/#installation) installed, kubectl installed

**Quick start** (using helper scripts):

```bash
./scripts/deploy.sh        # Create cluster, build, import, apply manifests
./scripts/status.sh         # Check pod/service/ingress status
./scripts/rebuild.sh        # Rebuild images and restart deployments
./scripts/logs.sh           # View pod logs
./scripts/clean.sh          # Delete cluster
```

**Manual steps**:

```bash
# 1. Create k3d cluster with port mapping
k3d cluster create sdwc -p "8080:80@loadbalancer"

# 2. Build Docker images (from project root)
docker build -f sdwc-api/Dockerfile -t ghcr.io/seongmin15/sdwc/sdwc-api:latest .
docker build -f sdwc-web/Dockerfile -t ghcr.io/seongmin15/sdwc/sdwc-web:latest .

# 3. Import images into k3d
k3d image import ghcr.io/seongmin15/sdwc/sdwc-api:latest ghcr.io/seongmin15/sdwc/sdwc-web:latest -c sdwc

# 4. Apply Kubernetes manifests
kubectl apply -f infra/sdwc-api/deployment.yaml
kubectl apply -f infra/sdwc-web/deployment.yaml
kubectl apply -f infra/ingress.yaml

# 5. Verify
kubectl get pods            # Both pods should be Running
curl http://localhost:8080/health   # {"status":"ok"}
curl http://localhost:8080/         # SDwC web UI
```

Access at **http://localhost:8080**

To shut down: `k3d cluster delete sdwc`
